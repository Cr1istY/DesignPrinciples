### 什么是里氏替换原则

继承必须确保超类（父类）所拥有的性质在子类中仍然成立。

简单来讲，里氏替换原则，即，子类一定能实现父类的全部功能。

例如，对于一个Bird类来说，如果，其中有一个接口为fly。

则，继承该类的子类必须能够重写fly方法。

里氏替换原则在理解上来说，是一个比较简单的原则。

不过，在实际编码中，由于项目的复杂性，我们有时会忘记改原则。

#### 四点含义

1. 子类可以实现父类的抽象方法，     
2. 子类可以增加自己特有的方法
3. 当子类的方法重载父类的方法时，方法的前置条件（即方法的形参）要比父类方法的输入参数更宽松。
4. 当子类的方法实现父类的方法（重写、重载或实现抽象方法时），方法的后置条件要比父类的方法更严格或与父类的方法相等。
